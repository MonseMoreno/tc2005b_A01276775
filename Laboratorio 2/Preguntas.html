¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet se trata de una serie de computadoras conectadas enté sí, las cuales conforman una red de comunicación mundial. Por otro lado la World Wide Web es un sistema que funciona atraves de las computadoras interconectadas (Internet) por el cual se pueden transmitir diversos tipos de datos a través de HTTP (Protocolo de Transferencia de Hipertextos ).

¿Cuáles son las partes de un URL?

Comenzando con por la definición URL por sus siglas significa Uniform Resource Locator
Y consta de varias partes que son:

  • PROTOCOLO HTTP o HTTPS Formato en que se van a enviar e intercambiar los datos que permitan cargar la página web
  • DOMINIO: Nombre de la dirección IP del servidor donde se almacena la página
  • CARPETA: Hace referencia al un fichero o archivo de la página web
  • PARÁMETROS: Datos adicionales que identifican o filtran un recurso en elservidor

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Realiza una petición a un recurso específico
  • HEAD: Funcionalmente hablando pide una respuesta similar al GET pero sin el cuerpo de la respuesta el servidor responde con líneas y headers.
  • POST: Enviar información/recursos al servidor.
  • PUT: Reemplaza todas las representaciones actuales del recurso, es decir sube un archivo.
  • PATCH: Realiza modificaciones parciales al recurso.
  • DELETE: Borra un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Al ingresarse por primera vez en el servidor se utilizaría el método POST, ya que, se tiene que registrar por primera vez en la base de datos y enviar un recurso.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El metodo GET, recupera la informacion de un lado y redirije o muestra al usuario es decir este método se usaría para solicitar al servidor que envíe un recurso de regreso

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No se trata de un error, realmente significa que la página a cargado correctamente

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Este tipo de errores con esos códigos de estado indican a los buscadores que la página no existe y que el contenido no se debería indexar aunque igual es importante que el desarrollador intente incluir algun mecanismo que permita a los usuarios avisarte si hay algún enlace roto.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

En este caso el desarrollador sí sería responsable de corregir el sitio web ya que el error 500 significa que ha sucedido un error al intentar acceder al servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Un atributo desaprobado o despreciado es aquel que ha quedado anticuado por la presencia de estructuras nuevas

  • acronym: Para abreviaciones se mantiene la etiqueta abbr
  • applet: Sustituida por object
  • isindex: Se utilizaba para mandar información al servidor dejó de ser funcional desde la aparición de los formularios.
  • dir: Para crear listas, actualmente recomienda usar ul

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • Se simplica el DOCTYPE: !DOCTYPE html
  • HTML5 permite incluir elementos de SVG y MathML.
  • Se introducen nuevos elementos, como: section, article, aside, header, footer, etc.
  • Se introducen nuevos atributos, como: media, charset, autofocus, placeholder, etc.

¿Qué componentes de estructura y estilo tiene una tabla?

  • Estructura: table
  • Leyenda: caption
  • Cuerpos: tbody, thead, tfoot
  • Filas: tr
  • Celdas: td, th
  • Columnas: col, colgroup
